# 인스타그램 발행 사이클 점검 — 분기 C 실행 보고

**실행 시각**: 2026-06-10 (수요일)
**자동화**: `instagram-publishing-cycle.prompt.md` → 분기 C (목·일 09:00 발행 결과 점검)
**결과**: ⛔ **실행 불가 (환경 차단)** — Drive 접근 경로 없음

---

## 요약

분기 C는 전적으로 Google Drive 폴더(`3-발행완료/`, `4-발행실패/`, `5-인사이트/`)를 읽어 결과를 점검하는 작업이다. 이번 실행 환경(Claude 샌드박스)에는 **Drive에 닿을 수단이 전혀 없어** Step C-1~C-3을 수행할 수 없었다. 따라서 실데이터가 없어 Step C-4 텔레그램 보고는 **발송하지 않았다** (수치를 지어내 보내는 것은 룰 위반).

| 항목 | 상태 |
|------|------|
| 필수 환경변수 7종 (`MUNGMUNG_BOT_TOKEN` 외) | ✅ 모두 SET |

→ 샌드박스에는 사용자 머신의 gdrive OAuth 설정이 없으므로, 이 환경에서 `rclone config`를 새로 깔아도 `gdrive:` remote에 접근할 수 없음.

## 날짜 불일치 메모

스케줄 작업 파일은 "오늘은 목요일 또는 일요일"을 전제하지만, 실제 실행일은 **수요일(2026-06-10)**이다. 분기 C는 cron `0 9 * * 4,0`(목·일)에만 도는 분기이므로, 오늘 트리거 자체가 스케줄과 어긋난다. 수요일에 의미 있는 분기는 A(화 18:00 생성분의 수요일 발행)이지 C가 아니다.

- 어제(2026-06-09, 화) — 발행일 아님
- 그저께(2026-06-08, 월) — 인사이트 대상 아님

설령 Drive에 닿았더라도 점검할 "전날 발행분"이 존재하지 않을 가능성이 높다.

## 점검 불가 항목 (Drive 필요)

- **C-1 발행 완료 확인**: `3-발행완료/`에서 어제 날짜 검색 — 불가
- **C-2 실패 케이스**: `4-발행실패/` 및 `_error.txt` 분석 — 불가
- **C-3 24h 인사이트**: `5-인사이트/`에서 그저께 발행분 수집 여부 — 불가

## 권장 조치

이 점검은 **Drive RW가 되는 환경에서 실행해야** 정상 동작한다. 둘 중 하나 필요:

1. **사용자 머신에서 직접 실행** — rclone + `gdrive:` remote가 설정된 로컬에서:
   ```bash
   claude --automation .claude/automations/instagram-publishing-cycle.prompt.md
   ```
2. **Google Drive MCP 커넥터 연결** — 샌드박스/Cowork에서 Drive 폴더를 읽도록 커넥터를 붙이면 rclone 없이도 분기 C 수행 가능.

추가로, **분기 C cron(목·일)과 실제 트리거 요일을 일치**시킬 것. 오늘처럼 수요일에 분기 C가 도는 것은 스케줄 오설정 신호일 수 있음.
